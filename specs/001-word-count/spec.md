# Feature Specification: 字數統計工具

**Feature Branch**: `001-word-count`  
**Created**: 2026-06-30  
**Status**: Draft  
**Input**: User description: "實作一個字數統計的功能，一個 textarea 讓使用者貼上（也有貼上按鈕）貼上後馬上在下方顯示出目前字數的統計。另外也有一鍵清除的功能"

## Clarifications

### Session 2026-06-30

- Q: 中文/CJK 文字的「字數」應如何計算？ → A: 混合計算——CJK 字元每字算 1 個「字」，非 CJK 內容以空白分隔的詞段計算，兩者相加。
- Q: 輸入區已有文字時，「貼上」按鈕如何處理剪貼簿內容？ → A: 取代輸入區全部內容。
- Q: v1 要顯示哪些統計項目？ → A: 四項全包——字元數、不含空白字元數、字數、行數。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 貼上文字並即時看到字數統計 (Priority: P1)

使用者把一段文字（例如文章、訊息草稿）貼進工具的輸入區，或按下「貼上」按鈕從剪貼簿帶入內容。內容一進入輸入區，工具立刻在下方更新並顯示目前的字數統計，不需要額外按「計算」。

**Why this priority**: 這是整個工具的核心價值——即時知道內容有多少字。沒有它，工具就沒有存在意義。

**Independent Test**: 在輸入區貼入或輸入一段已知字數的文字，確認下方統計數字立即且正確地反映內容，即可獨立驗證並交付價值。

**Acceptance Scenarios**:

1. **Given** 輸入區為空、統計顯示為 0，**When** 使用者在輸入區輸入或貼上文字，**Then** 下方統計即時更新為對應的字數。
2. **Given** 使用者已授權讀取剪貼簿且剪貼簿內有文字，**When** 使用者按下「貼上」按鈕，**Then** 剪貼簿文字被帶入輸入區並立即觸發統計更新。
3. **Given** 輸入區已有文字，**When** 使用者繼續編輯（新增或刪除字元），**Then** 統計隨每次變更即時重新計算。

---

### User Story 2 - 一鍵清除內容 (Priority: P2)

使用者完成一次統計後，想重新開始或清掉敏感內容，按下「清除」按鈕即可一次清空輸入區，統計同步歸零。

**Why this priority**: 提升重複使用的效率與隱私保障，但屬於核心統計功能之上的便利性，故次於 P1。

**Independent Test**: 在輸入區有內容的情況下按「清除」，確認輸入區被清空且統計歸零，即可獨立驗證。

**Acceptance Scenarios**:

1. **Given** 輸入區內有文字、統計顯示非 0，**When** 使用者按下「清除」按鈕，**Then** 輸入區被清空且所有統計數字歸零。
2. **Given** 輸入區已為空，**When** 使用者按下「清除」按鈕，**Then** 狀態維持為空且不產生錯誤。

---

### Edge Cases

- 輸入區為空時，統計應穩定顯示 0，而非空白或錯誤。
- 當瀏覽器拒絕或不支援剪貼簿讀取（例如未授權、非安全環境）時，「貼上」按鈕需給予清楚提示，並讓使用者仍可手動貼上（例如鍵盤貼上），不可讓工具卡住或無回饋。
- 貼上極大量文字（例如數萬字）時，統計仍需在可接受時間內完成，且介面不應凍結。
- 內容含換行、空白、emoji、中英混排與全形/半形字元時，統計需以一致且可預期的規則計算。
- 僅含空白或換行的內容，字元與字數的計算需符合定義（例如空白是否計入字元數）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 提供一個多行文字輸入區（textarea），讓使用者可手動輸入或貼上文字。
- **FR-002**: 系統 MUST 提供「貼上」按鈕，按下後從系統剪貼簿讀取文字並取代輸入區的全部現有內容。
- **FR-003**: 當剪貼簿讀取不被允許或失敗時，系統 MUST 顯示清楚的提示訊息，並讓使用者仍可透過手動方式貼上文字。
- **FR-004**: 當輸入區內容變更時（包含輸入、貼上、刪除），系統 MUST 即時、自動更新字數統計，無需使用者再按任何「計算」按鈕。
- **FR-005**: 系統 MUST 顯示以下四項統計：「字元數」（全部字元，含空白）、「不含空白字元數」、「字數（words）」、「行數」。字數採混合規則：每個 CJK 字元（中日韓文字）算 1 個字，非 CJK 內容以空白分隔的詞段計算，兩者相加。
- **FR-006**: 空輸入時，上述四項統計 MUST 均顯示為 0。
- **FR-007**: 系統 MUST 提供「清除」按鈕，一鍵清空輸入區並將所有統計歸零。
- **FR-008**: 統計顯示 MUST 位於輸入區下方。
- **FR-009**: 系統 MUST 將此工具註冊為一個可導覽的工具頁面，並在工具清單中標示為可用。
- **FR-010**: 所有使用者可見文字 MUST 使用繁體中文（zh-Hant），語氣與既有工具一致。
- **FR-011**: 所有互動元素（輸入區、貼上、清除按鈕、統計區）MUST 可用鍵盤操作，並具備可見的焦點指示與適當的無障礙標籤，符合 WCAG 2.2 AA。

### Key Entities *(include if feature involves data)*

- **輸入文字（Input Text）**: 使用者目前在輸入區中的內容，為統計計算的唯一來源；不需持久化儲存。
- **統計結果（Statistics）**: 由輸入文字即時衍生的計量值，包含字元數、不含空白字元數、字數與行數。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 使用者貼上或輸入文字後，統計數字在感知上即時更新（一般長度內容於 0.1 秒內反映變更）。
- **SC-002**: 對於已知字數的測試樣本，顯示的統計與預期計算規則 100% 相符。
- **SC-003**: 使用者能在 3 個步驟內（開啟工具 → 貼上 → 看到結果）完成一次字數統計。
- **SC-004**: 按下「清除」後，輸入區與所有統計於 1 秒內歸零，且 100% 不殘留先前內容。
- **SC-005**: 當剪貼簿不可用時，100% 的情況會顯示提示而非無聲失敗。

## Assumptions

- 此工具為純前端、即時計算，不需後端、不需登入、不需持久化儲存內容。
- 「字數」採混合計算：CJK 字元逐字計入，非 CJK 內容以空白分詞，兩者相加；因此純中文內容的字數約等同其字元數，符合 zh-Hant 使用者直覺。
- 「貼上」按鈕依賴瀏覽器剪貼簿讀取能力；在不支援或未授權的環境下，使用者仍可用鍵盤手動貼上作為退路。
- 視覺風格沿用 DESIGN.md 的像素/CRT 終端機美學與既有版面（左側工具清單 + 右側 main）。
- 行動裝置支援沿用既有響應式版面，不另立特例。
