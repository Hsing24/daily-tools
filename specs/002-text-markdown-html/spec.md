# Feature Specification: 文字轉 Markdown/HTML 工具

**Feature Branch**: `002-text-markdown-html`  
**Created**: 2026-07-10  
**Status**: Draft  
**Input**: User description: "Text to Markdown/HTML：將文字轉換為 Markdown 或 HTML 格式"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 貼上純文字並立即取得 Markdown 與 HTML 轉換結果 (Priority: P1)

使用者把一段純文字（例如筆記、草稿、多段落內容）貼進工具的輸入區。內容一進入輸入區，工具立刻在下方分別產生對應的 Markdown 格式與 HTML 格式輸出，保留原始的段落與換行結構，不需要使用者額外按「轉換」按鈕。

**Why this priority**: 這是工具的核心價值——把手邊的純文字快速變成可直接貼到 Markdown 文件或網頁中使用的格式。沒有它，工具就沒有存在意義。

**Independent Test**: 在輸入區貼入一段包含多個段落與換行的已知文字，確認下方同時出現正確對應的 Markdown 輸出與 HTML 輸出，即可獨立驗證並交付價值。

**Acceptance Scenarios**:

1. **Given** 輸入區為空、兩個輸出區塊皆為空，**When** 使用者在輸入區輸入或貼上文字，**Then** 下方的 Markdown 輸出與 HTML 輸出立即同步產生。
2. **Given** 輸入文字包含以空白行分隔的多個段落，**When** 轉換完成，**Then** Markdown 輸出以空白行分段、HTML 輸出以 `<p>` 區分各段落。
3. **Given** 輸入文字在同一段落內有單一換行，**When** 轉換完成，**Then** Markdown 輸出保留該換行（不與下一行合併成同一行）、HTML 輸出以 `<br>` 呈現對應換行。
4. **Given** 使用者已授權讀取剪貼簿且剪貼簿內有文字，**When** 使用者按下「貼上」按鈕，**Then** 剪貼簿文字取代輸入區內容並立即觸發轉換更新。

---

### User Story 2 - 複製其中一種輸出格式 (Priority: P2)

使用者看到轉換結果後，想把 Markdown 或 HTML 其中一種輸出整段複製到剪貼簿，貼到其他文件或程式中使用。

**Why this priority**: 轉換結果若無法方便地取出使用，價值就大打折扣，但仍建立在 P1 的轉換結果之上，故次於 P1。

**Independent Test**: 在任一輸出區塊按下「複製」按鈕，確認該區塊的完整內容被寫入剪貼簿並有清楚的成功提示，即可獨立驗證。

**Acceptance Scenarios**:

1. **Given** Markdown 輸出區塊已有內容，**When** 使用者按下該區塊的「複製」按鈕，**Then** 該內容被複製到剪貼簿，並顯示複製成功的提示。
2. **Given** 瀏覽器拒絕或不支援剪貼簿寫入，**When** 使用者按下「複製」按鈕，**Then** 系統顯示清楚提示，並讓使用者仍可手動選取文字複製。

---

### User Story 3 - 一鍵清除內容 (Priority: P3)

使用者完成一次轉換後，想重新開始或清掉內容，按下「清除」按鈕即可一次清空輸入區，兩個輸出區塊同步清空。

**Why this priority**: 提升重複使用的效率與隱私保障，屬於核心轉換功能之上的便利性，故優先度最低。

**Independent Test**: 在輸入區與兩個輸出區塊皆有內容的情況下按「清除」，確認三者同步清空，即可獨立驗證。

**Acceptance Scenarios**:

1. **Given** 輸入區與兩個輸出區塊皆有內容，**When** 使用者按下「清除」按鈕，**Then** 輸入區與兩個輸出區塊同步清空。
2. **Given** 輸入區已為空，**When** 使用者按下「清除」按鈕，**Then** 狀態維持為空且不產生錯誤。

---

### Edge Cases

- 輸入區為空或僅含空白/換行時，兩個輸出區塊應穩定顯示為空，而非錯誤或殘留內容。
- 段落之間出現連續多個空白行時，應視為單一段落分隔，不應產生多個空白段落。
- 輸入文字中恰好包含 Markdown 語法字元（例如 `*`、`_`、`#`、`` ` ``、`-`、`>`、`[`、`]`）或 HTML 標籤字元（`<`、`>`、`&`）時，轉換結果必須將其視為一般文字內容並適當跳脫，不可被誤判為既有語法或破壞輸出結構。
- 貼上極大量文字（例如數萬字）時，轉換仍需在可接受時間內完成，且介面不應凍結。
- 當瀏覽器拒絕或不支援剪貼簿讀取/寫入時，「貼上」與「複製」按鈕需給予清楚提示，並讓使用者仍可用鍵盤手動操作作為替代方案。
- 輸入文字開頭或結尾含有多餘空白行時，輸出不應產生多餘的空段落。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 提供一個多行文字輸入區（textarea），讓使用者可手動輸入或貼上純文字內容。
- **FR-002**: 系統 MUST 提供「貼上」按鈕，按下後從系統剪貼簿讀取文字並取代輸入區的全部現有內容；當剪貼簿讀取不被允許或失敗時，MUST 顯示清楚的提示訊息，並讓使用者仍可透過手動方式貼上文字。
- **FR-003**: 當輸入區內容變更時（包含輸入、貼上、刪除），系統 MUST 即時、自動重新產生 Markdown 格式輸出與 HTML 格式輸出，無需使用者另外按下「轉換」按鈕。
- **FR-004**: 轉換規則 MUST 以一個或多個連續空白行作為段落分隔：段落之間以空白行區隔；段落內的單一換行 MUST 視為同一段落內的換行並保留（Markdown 以行尾跳脫換行語法保留、HTML 以 `<br>` 保留）。
- **FR-005**: HTML 輸出 MUST 對輸入文字中的 HTML 特殊字元（`<`、`>`、`&`、`"`）進行跳脫，避免破壞輸出結構或造成腳本注入風險。
- **FR-006**: Markdown 輸出 MUST 對輸入文字中恰好與 Markdown 語法相同的字元或符號進行跳脫，避免使用者的原始純文字被誤判為 Markdown 語法（例如清單、標題、粗體、引用等）。
- **FR-007**: 系統 MUST 分別提供 Markdown 輸出與 HTML 輸出兩個獨立區塊同時顯示，且各自附有「複製」按鈕，讓使用者可個別複製其中一種格式的完整內容。
- **FR-008**: 複製操作成功或失敗時，系統 MUST 提供清楚的視覺回饋（例如「已複製」提示）；若瀏覽器不支援或拒絕剪貼簿寫入，MUST 提示使用者並允許手動選取複製作為替代方案。
- **FR-009**: 系統 MUST 提供「清除」按鈕，一鍵清空輸入區並使兩個輸出區塊同步清空。
- **FR-010**: 輸入為空時，兩個輸出區塊 MUST 均顯示為空，不顯示錯誤或多餘內容。
- **FR-011**: 系統 MUST 將此工具註冊為一個可導覽的工具頁面，並在工具清單中標示為可用。
- **FR-012**: 所有使用者可見文字 MUST 使用繁體中文（zh-Hant），語氣與既有工具一致。
- **FR-013**: 所有互動元素（輸入區、貼上、複製、清除按鈕、輸出區塊）MUST 可用鍵盤操作，並具備可見的焦點指示與適當的無障礙標籤，符合 WCAG 2.2 AA；輸出區塊內容更新與複製成功提示 MUST 透過適當的 live region（例如 `aria-live="polite"`）讓輔助技術使用者可感知。

### Key Entities *(include if feature involves data)*

- **輸入文字（Source Text）**: 使用者目前在輸入區中的純文字內容，為轉換的唯一來源；不需持久化儲存。
- **Markdown 輸出（Markdown Output）**: 由輸入文字即時衍生的 Markdown 格式結果，保留段落與換行結構。
- **HTML 輸出（HTML Output）**: 由輸入文字即時衍生的 HTML 格式結果，保留段落與換行結構並完成必要的字元跳脫。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 使用者貼上或輸入文字後，Markdown 與 HTML 輸出在感知上即時更新（一般長度內容於 0.5 秒內反映變更）。
- **SC-002**: 對於已知結構（多段落、換行、Markdown/HTML 特殊字元）的測試樣本，輸出的 Markdown 與 HTML 100% 正確保留原始段落與換行結構，且特殊字元 100% 不會被誤判為語法或破壞輸出結構。
- **SC-003**: 使用者能在 3 個步驟內（開啟工具 → 貼上文字 → 複製其中一種輸出）完成一次轉換並取得可用結果。
- **SC-004**: 按下「清除」後，輸入區與兩個輸出區塊於 1 秒內同步歸零，100% 不殘留先前內容。
- **SC-005**: 當剪貼簿讀取或寫入功能不可用時，100% 的情況會顯示清楚提示而非無聲失敗。

## Assumptions

- 輸入視為尚未包含有意義 Markdown 語法或 HTML 標籤的純文字；文字中若恰好出現 Markdown/HTML 特殊字元，一律視為一般文字內容並於輸出中適當跳脫，而非被解讀為既有語法。
- v1 僅處理「段落與換行」層級的結構轉換；不自動偵測或轉換清單、標題、粗體/斜體等進階 Markdown 結構（例如不會將以 `- ` 開頭的行自動視為清單項目）。
- 此工具為純前端、即時運算，不需後端服務、不需登入、不需持久化儲存輸入或輸出內容。
- 「貼上」與「複製」功能依賴瀏覽器剪貼簿讀寫能力；在不支援或未授權的環境下，使用者仍可用鍵盤手動輸入或選取文字複製作為替代方案。
- 視覺風格沿用 DESIGN.md 的像素/CRT 終端機美學與既有版面（左側工具清單 + 右側 main）。
- 行動裝置支援沿用既有響應式版面，不另立特例。
